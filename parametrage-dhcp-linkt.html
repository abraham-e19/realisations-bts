Contexte général

Durant mon stage chez Linkt, j'ai traité plusieurs tickets liés à la configuration DHCP sur des équipements réseau clients (routeurs Cisco). Voici deux interventions réelles réalisées à distance via SecureCRT.

Interventions réalisées

Tâche 1

Mise en place d'un relais DHCP

Un client avec un routeur Cisco C1117 et plusieurs VLANs. Les clients du VLAN 2 devaient obtenir leurs IPs depuis un serveur DHCP distant à 10.164.4.1. Le routeur devait relayer les requêtes DHCP broadcast vers ce serveur. 

enable
configure terminal
interface Vlan2
  ip helper-address 10.164.4.1
exit
write memory
show running-config | include helper-address
⚠️ Le routeur transforme correctement les trames DHCP broadcast en unicast vers 10.164.4.1. Problème rencontré : le serveur 10.164.4.1 n'était pas joignable — relais correctement configuré, mais l'accessibilité du serveur restait à résoudre côté client.
Tâche 2

Activation du serveur DHCP sur un routeur 4G

Un routeur 4G de formation ne délivrait aucune adresse IP aux équipements branchés. Mission : activer le serveur DHCP directement sur la partie LAN du routeur. 

enable
configure terminal
service dhcp
ip dhcp pool FORMATION
  network         192.168.1.0 255.255.255.0
  default-router  192.168.1.1
  dns-server      8.8.8.8 8.8.4.4
exit
ip dhcp excluded-address 192.168.1.1
interface Vlan1
  ip address 192.168.1.1 255.255.255.0
  no shutdown
exit
write memory
✅ Les équipements branchés reçoivent automatiquement une IP de la plage 192.168.1.0/24 — serveur DHCP opérationnel.

Synthèse comparative

Mode Commande clé Rôle

Serveur DHCP local ip dhcp pool Distribuer les adresses IP directement depuis le routeur

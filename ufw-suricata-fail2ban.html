Architecture — défense en profondeur

Les trois outils fonctionnent en couches complémentaires. UFW filtre le trafic en amont, Suricata analyse le contenu des paquets autorisés, et Fail2ban réagit en bannissant automatiquement les IPs identifiées comme malveillantes.

1 — UFW Pare-feu

UFW (Uncomplicated Firewall) est le premier rempart. Il applique une politique deny incoming par défaut et n'autorise que les ports explicitement ouverts.

$ sudo apt install ufw $ sudo ufw default deny incoming $ sudo ufw default allow outgoing $ sudo ufw allow 22 $ sudo ufw allow 80 $ sudo ufw enable $ sudo ufw status
Résultat : tout le trafic entrant est bloqué par défaut — seuls SSH (port 22) et HTTP (port 80) sont autorisés.

2 — Fail2ban Bannissement

Fail2ban lit les journaux système et réseau, détecte les comportements suspects (trop de tentatives ratées) et bannit automatiquement les IPs via UFW.

$ sudo apt install fail2ban $ sudo nano /etc/fail2ban/jail.local $ sudo systemctl restart fail2ban $ sudo fail2ban-client status $ sudo fail2ban-client status sshd $ sudo fail2ban-client status suricata
/etc/fail2ban/jail.local
[sshd] enabled = true port = 22 maxretry = 3 bantime = 10m backend = systemd [suricata] enabled = true filter = suricata logpath = /var/log/suricata/fast.log maxretry = 1 bantime = 600 findtime = 600 banaction = ufw

3 — Suricata IDS

Suricata est un système de détection d'intrusion (IDS) qui analyse le trafic réseau en temps réel et génère des alertes dans /var/log/suricata/fast.log. Ces alertes sont ensuite lues par Fail2ban pour déclencher les bans.

$ sudo apt install suricata $ sudo nano /etc/suricata/suricata.yaml $ sudo systemctl restart suricata $ sudo systemctl status suricata $ tail -f /var/log/suricata/fast.log

4 — Tests d'attaque depuis Windows

Pour valider la chaîne de détection, trois types d'attaques sont lancés depuis la machine Windows vers la VM Bastion. Trois terminaux sont ouverts simultanément pour observer les réactions en temps réel.

Terminal 1 — UFW
sudo ufw status
Terminal 2 — Suricata
tail -f fast.log
Terminal 3 — Fail2ban
fail2ban-client status
1 Scan de ports — Nmap
  • Lancement d'un scan Nmap depuis Windows sur l'IP de la VM Bastion
  • Suricata détecte le comportement de scan et génère une alerte dans fast.log
  • Fail2ban lit l'alerte via la jail suricata et bannit l'IP attaquante
2 Brute force SSH — 5 tentatives ratées
  • 5 tentatives de connexion SSH avec des mots de passe incorrects
  • Fail2ban (jail sshd) bannit l'IP après le 3ème échec (maxretry = 3)
  • Les tentatives suivantes sont bloquées directement par UFW — ban de 10 minutes
3 Flood HTTP — rafraîchissements rapides
  • Rafraîchissements très rapides sur le port 80 de la VM Bastion
  • Suricata identifie le flood HTTP et génère une alerte dans fast.log
  • L'IP est transmise à Fail2ban via la jail suricata pour bannissement

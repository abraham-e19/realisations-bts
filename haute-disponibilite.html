TP Haute Disponibilité HD1 & HD2 — Startup Seaboatonline

🔁 Clustering HA 🖥️ Proxmox / Debian 12 🗄️ Réplication BDD
Contexte
La startup Seaboatonline gère la location de bateaux en ligne. Le gérant M. Dupont/Boussard exige une disponibilité 24h/24 du site web et de la base de données — 2 VMs Debian hébergées sous Proxmox.
Outils
Debian 12 — Apache2 — Corosync — Pacemaker — MariaDB — Adminer
Période
2ème année BTS SIO SISR — 2025-2026

Architecture générale

Le projet se divise en deux volets complémentaires : HD1 assure la haute disponibilité du service web via un cluster Corosync / Pacemaker avec basculement automatique, et HD2 assure la continuité de la base de données MariaDB par réplication Master / Slave entre les deux nœuds.

4 — Désactivation STONITH Cluster config

STONITH (Shoot The Other Node In The Head) est le mécanisme de fencing du cluster. En environnement de test sans matériel dédié, il est désactivé pour permettre au cluster de démarrer correctement.

$ crm_verify -L -V $ crm configure property stonith-enabled=false $ crm configure show
Résultat : crm configure show confirme stonith-enabled=false ✅

8 — Réplication Master / Slave Master Slave

BDD1 (Node1 — 10.0.1.112) est configurée en tant que Master. BDD2 (Node2 — 10.0.1.113) est configurée en tant que Slave. Toute écriture sur le Master est automatiquement répliquée vers le Slave, assurant la continuité des données en cas de bascule.

  • Activer les logs binaires sur le Master (log_bin dans my.cnf)
  • Créer l'utilisateur de réplication sur le Master
  • Effectuer un dump de la base Master (mysqldump)
  • Importer le dump sur le Slave
  • Configurer le Slave : CHANGE MASTER TO (host, user, password, log file, position)
  • Démarrer la réplication : START SLAVE;
$ # Sur le Master — BDD1 MariaDB> CREATE USER 'repl'@'10.0.1.113' IDENTIFIED BY 'replpass'; MariaDB> GRANT REPLICATION SLAVE ON *.* TO 'repl'@'10.0.1.113'; MariaDB> SHOW MASTER STATUS; $ # Sur le Slave — BDD2 MariaDB> CHANGE MASTER TO MASTER_HOST='10.0.1.112', MASTER_USER='repl', MASTER_PASSWORD='replpass', MASTER_LOG_FILE='...', MASTER_LOG_POS=...; MariaDB> START SLAVE; MariaDB> SHOW SLAVE STATUS\G

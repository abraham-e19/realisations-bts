Période
2ème année BTS SIO SISR — 2025-2026

Contexte

L'ESN Novatice édite des logiciels destinés aux médiathèques. Elle dispose d'un serveur Windows et d'un serveur Linux au sein de son infrastructure.

La mission est de configurer un serveur DNS faisant autorité sur le domaine novatice.fr sur les deux systèmes, avec zones directe et inverse, enregistrements A, PTR et CNAME.

Partie 1 — Linux

Debian 12 + Bind9

Installer et configurer Bind9 comme serveur DNS faisant autorité sur novatice.fr.

Étapes réalisées

  1. Installation du paquet bind9 sur Debian
  2. Configuration du fichier named.conf.local avec zone directe et zone inverse
  3. Création du fichier de zone directe dans /var/cache/bind/novatice
  4. Création du fichier de zone inverse dans /var/cache/bind/novatice.rev
  5. Test nslookup zone directe : debian11.novatice.fr → 192.168.0.72
  6. Test zone inverse : 192.168.0.72 → debian11.novatice.fr
  7. Test client : accès à www.novatice.fr depuis un navigateur

Partie 2 — Windows

Windows Server 2019 — Gestionnaire DNS

Configurer le rôle DNS sur Windows Server 2019 via le Gestionnaire DNS.

Enregistrements configurés

Nom Type Valeur
www Hôte A 192.168.0.95
commercial Alias CNAME www.novatice.fr
compta Alias CNAME www.novatice.fr
rh Alias CNAME www.novatice.fr
tech Alias CNAME www.novatice.fr

Étapes réalisées

  1. Installation du rôle DNS via le Gestionnaire de serveur
  2. Création de la zone directe novatice.fr
  3. Ajout des enregistrements : www (Hôte A → 192.168.0.95), commercial / compta / rh / tech (Alias CNAME → www.novatice.fr)
  4. Création de la zone inverse 0.168.192.in-addr.arpa avec pointeurs PTR
  5. Test PowerShell : nslookup www.novatice.fr et nslookup 192.168.0.95
  6. Test navigateur : www.novatice.fr affiche la page IIS

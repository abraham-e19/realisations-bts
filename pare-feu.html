1 — Politique par défaut

Avant toute règle spécifique, Windows Defender Firewall applique une politique globale sur le trafic entrant et sortant. La configuration retenue pour ce TP est la suivante :

Trafic entrant
🚫 Bloquer par défaut
Trafic sortant
✅ Autoriser par défaut

Le tableau ci-dessous récapitule l'ensemble des règles appliquées sur le poste 192.168.0.75.

N° IP source Port src IP dest Port dest Protocole Action Description

5 * * 192.168.0.75 20-21 FTP Autoriser FTP
6 * * 192.168.0.75 3389 RDP Autoriser Bureau à distance
7 * * 192.168.0.75 80 HTTP Autoriser Serveur web IIS

3 — ICMP / Ping

  • Protocole ICMP — pas de port spécifique (port 1 par convention)
  • Règle préexistante désactivée : Diagnostics réseau de base – Demande d'écho ICMPv4
  • Activation de la règle → le ping fonctionne ✅

4 — Installation des services réseau

  • 3389 RDP Bureau à distance — activé via Paramètres → Système → Bureau à distance
  • 80 HTTP IIS — installé via Panneau de configuration → Activer ou désactiver des fonctionnalités Windows
  • 20-21 TCP FTP — Serveur FTP installé via la fonctionnalité IIS
